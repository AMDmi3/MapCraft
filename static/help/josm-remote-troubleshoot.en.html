Ups! Failed to send remote command to the JOSM

How to find the cause:
  • Make sure that JOSM is opened. If not, then run it and try again.
  • Go to JOSM preferences (F12), open tab 'Remove Control' and make sure that this function is enabled. If not, enable it and try again.
  • If you have Firefox with addon NoScript installed, then go to the plugin's preferences, tab 'Advanced' and look for tab ABE. If there is one, then open it. This function by default prevents external requests to local sites. You can disable it or add following rules into "USER" section: 
    	  Site 127.0.0.1:8111
	  Accept
    Then hit button 'Refresh', then 'OK' and try again.
Nothing helps? Go to issue tracker and search for similar bug. Try enter word 'Remote' into search box. May be there is a solve.

If there are no one, then try to localize the bug: switch browser, sniff network traffic on port 127.0.0.1:8111 and so on. Then fill new issue. Thank you.
